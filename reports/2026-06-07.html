Gemini Live 支持实时创建与编辑图像

来源：X · @GeminiApp
Google 把 Gemini Live 从「对话」推向「实时共创」：用户开启 Live 按钮、共享摄像头后，可以边看画面边口述想要的修改，模型在像素级即时重绘。从房间布置预览到数学题手写板书，再到可分享的梗图，整条链路从「先描述—再等待—最后看结果」压缩到「一边说一边看见」。这意味着多模态实时生成首次进入大众消费级默认入口，Google 用一个产品功能把「AIGC 工具」与「日常对话助手」的边界抹掉了。
把生成式 AI 从「出图网站」拽进聊天框，下一个被颠覆的是 Midjourney 和绘图 App 的护城河。

GitHub 开源 Spec Kit：先把规范写好，再让 AI 写代码

来源：X · Rohan Paul
GitHub 推出的 Spec Kit 是一套开源工具链，专门治理「vibe coding」的最大病灶——AI 在需求没敲定前就匆忙动笔。它把流程倒过来：先写产品规范、澄清差距、制定技术计划、拆解任务，最后才让 agent 落地。规范本身成为一份可执行的开发合约，已经兼容 Copilot、Claude Code、Codex 和 Gemini CLI。
真正的 spec-driven development，不是写更好的 prompt，而是把规范当成一等公民编译进工具链。

Claude 是否在 rsync 中制造了更多 bug？

来源：Hacker News · alexispurslane.github.io
一篇 105 分热度的 HN 帖子借数据分析提出疑问：在 Claude 等代码助手进入维护者工作流之后，开源工具 rsync 的 bug 报告节奏是否发生了变化，并附上完整分析链接。这并非指控「AI 写错代码」，而是审视「AI 协助提交的 patch 是否带来了新的失败模式」——是当下值得建立系统性基线的问题。
真正的麻烦不是「AI 写错代码」，而是「AI 写出看起来对、但风格漂移的代码」淹没了 reviewer。
